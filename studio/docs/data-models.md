# 🛡️ Data Models & Schema Reference 🛡️

> **Authoritative JSON contracts and state persistence schemas**
> Rigid data validation schemas governing LND Studio's agentic loop and preventing AI hallucinations.

---

## 🔒 The Principle of Strict Schema Locking

To eliminate the unpredictability inherent in LLM generation, LND Studio mandates that all inter-agent communications are formatted as strict JSON payloads. Every schema is configured with `"additionalProperties": false` recursive constraints. 

If an LLM attempts to hallucinate fields not explicitly defined in the schema, the parser rejects the payload, forcing a regeneration. This guarantees that all downstream components receive perfectly structured data.

## 📋 The 4 Authoritative JSON Contracts

The schema definition files reside in `studio/schemas/` and are structured as follows:

### 1. `forensic-state.schema.json`
* **Purpose:** Generated by Kana (`KAN`) and consumed by Suki (`SUK`). Encapsulates all physical and text-based details extracted from the manga panel source.
* **Core Fields:**
  ```json
  {
    "page_number": "integer",
    "panels": [
      {
        "panel_id": "integer",
        "characters": [
          {
            "name": "string",
            "position": "string (e.g. left, center-right)",
            "clothing_state": "string (e.g. dressed, disheveled, naked)",
            "arousal_markers": "array of strings"
          }
        ],
        "bodily_fluids": "array of strings",
        "speech_balloons": [
          {
            "speaker": "string",
            "raw_japanese": "string",
            "translation_stub": "string"
          }
        ],
        "sfx": [
          {
            "raw_sfx": "string",
            "suggested_meaning": "string"
          }
        ]
      }
    ]
  }
  ```

### 2. `draft-prose.schema.json`
* **Purpose:** Generated by Suki (`SUK`) and audited by Riko (`RIK`).
* **Core Fields:**
  ```json
  {
    "page_number": "integer",
    "narrative_pov": "string (enforces strict third-person or first-person limits)",
    "paragraphs": "array of strings (the actual Vietnamese prose blocks)",
    "word_count": "integer",
    "sensory_check": {
      "has_sight": "boolean",
      "has_sound": "boolean",
      "has_touch": "boolean",
      "has_scent": "boolean"
    },
    "sfx_integrated": "array of strings (Romaji sounds integrated into paragraphs)"
  }
  ```

### 3. `audit-report.schema.json`
* **Purpose:** Generated by Riko (`RIK`) and consumed by Director K (`DIR`) to determine loop completion.
* **Core Fields:**
  ```json
  {
    "audit_passed": "boolean (true if score >= 85)",
    "score": "integer (0-100)",
    "categories": {
      "canon_adherence": "integer (0-20)",
      "sensory_density": "integer (0-20)",
      "character_consistency": "integer (0-20)",
      "linguistic_flow": "integer (0-20)",
      "dialogue_fidelity": "integer (0-20)"
    },
    "violations": [
      {
        "category": "string",
        "description": "string",
        "violating_text": "string"
      }
    ],
    "actionable_feedback": "string (concrete rewrite instructions on failure)"
  }
  ```

### 4. `continuity-ledger.schema.json`
* **Purpose:** Written to the continuity ledger by Orion (`ORI`) on successful audit to maintain state across page boundaries.
* **Core Fields:**
  ```json
  {
    "last_updated_page": "integer",
    "character_arousal_levels": {
      "character_name": "integer (0-100)"
    },
    "clothing_states": {
      "character_name": "string"
    },
    "current_location": "string",
    "active_plot_threads": "array of strings"
  }
  ```

## 🔄 State Persistence: `state.yaml`

To support resilient resume capabilities, Director K maintains a session-level state file at `/home/datdang/working/lnd_dev/studio/resources/state.yaml`:

```yaml
session_id: "065a0807-50ab-4fd3-a765-6f0e75a6dfc5"
project_name: "lnd_manga_vol_1"
status: "in_progress"
last_completed_page: 24
current_step: "step_4_prose_generation"
active_agents:
  - "DIR"
  - "SUK"
history:
  - page: 24
    step: "step_5_quality_audit"
    status: "completed"
    timestamp: "2026-05-22T19:54:12Z"
    score: 88
```
Using this schema, Director K immediately reads the state file on boot, loads the current context, and jumps directly to the unfinished step.
