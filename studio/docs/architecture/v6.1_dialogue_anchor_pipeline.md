# LND Studio V6.1 Pipeline Architecture (Dialogue-Anchor Protocol)

This document visualizes the execution flow of the **Manga Adaptation Pipeline** handled by the `manga-adapter` (Kana), detailing the core processes inside the `panel-forensic` (Prof. Atomic) and `lewd-writer` (Suki) modules.

## Architecture Change Summary (V6.1)

- **Panel Forensic** was shifted from purely visual scanning to a **Dialogue-First** approach (Pure OCR) to prevent AI hallucination, effectively anchoring physical and environmental details to parsed text.
- **Lewd Writer** was shifted from writing separate Action Prose and Dialogue Integration steps to a unified **Dialogue-Driven Action Prose** step, following a mandatory _Pre-speech -> Dialogue Anchor -> Post-speech_ sequence.

### Core Interactions to Maintain

1. **The OCR Safety Gate:** `panel-forensic` MUST execute Pure OCR (Step 2) before attempting any visual analysis. This is non-negotiable and prevents the AI from substituting real action with hallucinated R18 tropes.
2. **The "Chain Reaction" Structure:** Maintenance edits to `lewd-writer` must never break the fundamental trinity of the Dialogue-Driven Action step: _Foreplay (Setup) -> Anchor (Dialogue) -> Betrayal (Physical Reaction)._
3. **Sensory Verification:** Step 5c relies on the outputs generated in Step 3 and Step 4. If those steps fail to output sensory nouns (smell, touch, hearing), Step 5c will trigger a failure flag.
