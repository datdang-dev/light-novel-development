# RPG Game Adaptation Pipeline (V1)

**Goal:** Adapt an RPG Maker game into Light Novel prose by triangulating a Full-Save Playthrough Log against the game's complete Translation Script to recover lost narrative context, then delegate world-building, character profiling, and per-heroine novel generation to specialist agents.

**Orchestrator:** Director K (lnd-orchestrator).

---

## INPUTS (User MUST provide)

| #   | File                       | Role                   | Description                                                                                                                                     |
|-----|----------------------------|------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------|
| 1   | `playthrough_log.txt`      | **Anchor**             | Generated by `TimelineLogger.js`. Contains timestamped dialogue, choices, map transitions, and event triggers from a Full-Save CG playthrough. |
| 2   | `manual_translation.json`  | **Context Dictionary** | Contains ALL translatable text strings in the game. Used to find buildup/aftermath dialogue NOT captured by the log.                            |
| 3   | `{game_root_path}`         | **Game Root**          | Absolute path to the game's root directory (the folder containing `www/` or equivalent). User provides this at initialization.                  |

---

## WORKFLOW PHASES

### PHASE 1: Context Routing (Data Extraction & Cross-Reference)

> **рҹ”„ [Self-Executed by Director K]**

- **Step 1:** Initialize, Validate Game Structure & Inputs (`steps/step-01-initialize.md`)
  - *Action 1:* Ask user for `{game_root_path}`, `{game_name}`, and paths to `playthrough_log.txt` + `manual_translation.json`.
  - *Action 2:* Run **Prerequisite Validation Checklist** (see above). STOP on any HARD failure. WARN on SOFT failures.
  - *Action 3:* Scan `www/data/Map*.json` to build a **Map Index** (`map_id вҶ’ map_name`) for later context routing.
  - *Action 4:* Create output directory structure.
  - *Output:* `_lnd-output/_rpg/{game_name}/` + `map_index.json`

- **Step 2:** Context Extraction & Triangulation (`steps/step-02-context-routing.md`)
  - *Action:* Parse `playthrough_log.txt` to identify all triggered scenes/events. For each scene, search `manual_translation.json` to find the surrounding dialogue (buildup/aftermath) that was NOT in the log.
  - *Output:* `_lnd-output/_rpg/{game_name}/context_report.md`

- **вҸёпёҸ CHECKPOINT: Human Audit**
  - *Action:* Present `context_report.md` to the Boss for review and approval.
  - *Gate:* Do NOT proceed to Phase 2 until the Boss explicitly approves.

---

### PHASE 2: World & Character Foundation

- **Step 3:** World Info Extraction
  - *Input:* `context_report.md` + `manual_translation.json`
  - *Output:* `_lnd-output/_rpg/{game_name}/world_info.md`
  - **рҹ”„ [Delegating to Luna (world-weaver)]**
  - *Protocol:* Extract world systems, locations, factions, magic/technology rules.

- **Step 4:** Character Bible Generation
  - *Input:* `context_report.md` + `world_info.md`
  - *Output:* `_lnd-output/_rpg/{game_name}/characters/{character_name}_bible.md`
  - **рҹ”„ [Delegating to Aria (character-architect)]**
  - *Protocol:* Generate a Character Bible for the protagonist and each major heroine. Include personality, speech patterns, relationships, and arc progression.

- **вҸёпёҸ CHECKPOINT: Human Audit**
  - *Action:* Present generated `world_info.md` and all `character bibles` for Boss review.
  - *Gate:* Do NOT proceed to Phase 3 until the Boss explicitly approves.

---

### PHASE 3: Character-Focused Novelization (LOOP)

> This phase runs as a **repeating loop** per heroine.

- **Step 5:** Character Selection
  - *Action:* Present a list of available heroines (extracted from Phase 2 bibles) and ask the Boss: *"Which heroine's storyline do you want to develop next?"*
  - *Gate:* Wait for Boss selection. If Boss says "Done" or "Exit", terminate the loop.

- **Step 6:** Story Timeline Assembly
  - *Input:* Selected heroine's `bible.md` + `context_report.md`
  - *Output:* `_lnd-output/_rpg/{game_name}/novels/{heroine_name}/timeline.md`
  - **рҹ”„ [Self-Executed by Director K]**
  - *Protocol:* Filter the full context report to extract ONLY the scenes, dialogues, and events relevant to the selected heroine. Arrange them chronologically into a story timeline.

- **Step 7:** Prose Generation
  - *Input:* `timeline.md` + heroine's `bible.md` + `world_info.md`
  - *Output:* `_lnd-output/_rpg/{game_name}/novels/{heroine_name}/chapters/ch{NN}_{title}.md`
  - **рҹ”„ [Delegating to Suki (lewd-writer)]**
  - *Protocol:* Generate Light Novel prose chapters following Suki standards (metadata block, 3-layer scent protocol, Romanized SFX, continuity table, Vietnamese titles). Each significant scene/event becomes one chapter.

- **Step 8:** Quality Audit
  - *Input:* Generated chapter `draft.md`
  - *Output:* `audit.json`
  - **рҹ”„ [Delegating to Riko (gooner-editor)]**
  - *Logic:* If Score < 85 вҶ’ **Rewind to Step 7** with `fix` instructions.

- **вҸёпёҸ CHECKPOINT: Loop Decision**
  - *Action:* Return to **Step 5** and ask: *"Heroine {name}'s storyline is complete. Write for another heroine?"*

## CRITICAL INSTRUCTIONS

1. **рҹ”„ DELEGATION ICON:** Every step that delegates to a specialist agent MUST display the `рҹ”„ [Delegating to {AGENT_NAME}]` icon. This is a HARD REQUIREMENT to prevent hallucination and maintain clear agent boundaries. Steps executed by Director K directly use `рҹ”„ [Self-Executed by Director K]`.
2. **ARTIFACTS:** All intermediate files MUST be saved to the structure: `_lnd-output/_rpg/{game_name}/`.
3. **HUMAN GATES:** There are 3 mandatory human audit checkpoints (after Phase 1, after Phase 2, and the Loop Decision in Phase 3). NEVER skip these.
4. **CONTEXT ISOLATION:** Each specialist agent receives ONLY its designated inputs. Do NOT pass the full `manual_translation.json` to Suki. Suki receives the pre-processed `timeline.md` and `bible.md`.
5. **SUKI STANDARDS:** All prose output MUST comply with Suki's R18 writing standards as defined in the studio rules.
