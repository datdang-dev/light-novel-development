# RPG Game Adaptation Pipeline (V2)

**Goal:** Adapt an RPG Maker MV/MZ game into Light Novel prose by extracting character events, world data, and dialogue directly from game data files, then delegating world-building, character profiling, and per-heroine novel generation to specialist agents.

**Orchestrator:** Director K (lnd-orchestrator).

---

## INPUTS (User provides)

| #   | File                      | Required? | Description                                                                |
|-----|---------------------------|-----------|----------------------------------------------------------------------------|
| 1   | `{game_root_path}`        | ✅ HARD   | Absolute path to the game's root directory                                 |
| 2   | `{game_name}`             | ✅ HARD   | Name identifier for this adaptation project                                |
| 3   | `playthrough_log.txt`     | ⚠️ SOFT   | Optional. Generated by TimelineLogger.js. Enables log-assisted mode.       |
| 4   | `ManualTransFile.json`    | ⚠️ SOFT   | Optional. Translation dictionary for additional context.                   |

---

## ENGINE AUTO-DETECTION

The pipeline auto-detects the engine version:

```text
IF game.rmmzproject exists → RPG Maker MZ (flat: data/)
ELIF Game.rpgproject exists → RPG Maker MV (www: www/data/)
ELIF www/data/ exists      → RPG Maker MV (deployed)
ELIF data/ exists          → Generic RPG Maker (flat fallback)
```

---

## OPERATING MODES

| Mode | Trigger | Description |
|------|---------|-------------|
| **Data-Only** (default) | No playthrough log | Extracts directly from Map*.json + CommonEvents.json |
| **Log-Assisted** | User provides log | Uses log for event ordering + triangulates with translation |

---

## SCRIPTS

| Script | Purpose | Usage |
|--------|---------|-------|
| `validate_game_structure.py` | Auto-detect engine, validate structure, report stats | `python3 validate_game_structure.py {game_root}` |
| `build_map_index.py` | Build map lookup from MapInfos.json | `python3 build_map_index.py {data_dir} -o map_index.json` |
| `extract_character_events.py` | **Core** — extract all events for a character | `python3 extract_character_events.py {game_root} --actor-id N --actor-name NAME` |
| `parse_playthrough_log.py` | Parse TimelineLogger logs (log-assisted mode) | `python3 parse_playthrough_log.py {log} --map-index map_index.json` |

## WORKFLOW PHASES

### PHASE 1: Initialization & Data Extraction

> **🔄 [Self-Executed by Director K]**

- **Step 1:** Initialize & Validate (`steps/step-01-initialize.md`)
  - Auto-detect engine (MV/MZ)
  - Validate game structure (HARD/SOFT checks)
  - Build map index
  - Select operating mode (Data-Only / Log-Assisted)
  - Report: engine, maps, actors, CEs, mode

- **Step 2:** Context Extraction & Routing (`steps/step-02-context-routing.md`)
  - **Data-Only:** Run `extract_character_events.py` for each target character
  - **Log-Assisted:** Parse log + triangulate with translation + merge with extraction
  - Output: `{actor_name}_events.md` + `.json` per character
  - Compile context report with NSFW scene catalog

- **⏸️ CHECKPOINT: Human Audit**
  - Present extraction results + context report to user
  - Do NOT proceed until user approves

---

### PHASE 2: World & Character Foundation

- **Step 3:** World Info Extraction (`steps/step-03-world-info.md`)
  - **🔄 [Delegating to Luna (world-weaver)]**
  - Input: Extracted events + ManualTransFile.json
  - Output: `world_info.md`

- **Step 4:** Character Bible Generation (`steps/step-04-character-bibles.md`)
  - **🔄 [Delegating to Aria (character-architect)]**
  - Input: `{actor_name}_events.json` + introduction CE + erosion data
  - Output: `{actor_name}_bible.md`

- **⏸️ CHECKPOINT: Human Audit**
  - Present `world_info.md` + character bibles for user review

---

### PHASE 3: Character-Focused Novelization (LOOP)

- **Step 5:** Heroine Selection (`steps/step-05-heroine-select.md`)
  - Present heroines list → user picks → or "Done" to exit

- **Step 6:** Story Timeline Assembly (`steps/step-06-timeline.md`)
  - **🔄 [Self-Executed by Director K]**
  - Filter events for selected heroine → arrange chronologically

- **Step 7:** Prose Generation (`steps/step-07-prose.md`)
  - **🔄 [Delegating to Suki (lewd-writer)]**
  - Generate chapters following R18 standards

- **Step 8:** Quality Audit (`steps/step-08-quality-audit.md`)
  - **🔄 [Delegating to Riko (gooner-editor)]**
  - Score < 85 → rewind to Step 7

- **⏸️ CHECKPOINT: Loop Decision**
  - Return to Step 5 for next heroine or exit
